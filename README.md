## 🚀 Usage

### 1. Add your public SSH key

Edit `terraform/terraform.tfvars` and replace the `ssh_public_key` line with your actual key:

```bash
cat ~/.ssh/id_ed25519.pub   # or id_rsa.pub
# paste the result into terraform.tfvars
```

If you don't have one:

```bash
ssh-keygen -t ed25519 -C "your_user@your_machine"
```

### 2. Run Terraform

```bash
cd terraform/
terraform init
terraform plan
terraform apply
```

This will:
- Download the Fedora 43 LXC template to the `local` storage
- Download the Fedora 43 cloud image to the `local` storage
- Create the 3 LXC containers (VMIDs 201–203) with hostname, static IP, and root SSH enabled
- Create the 2 VMs (VMIDs 301–302) with cloud-init (hostname, static IP, root SSH enabled, qemu-guest-agent)
- Generate the Ansible inventory at `ansible/inventory/hosts.ini`

### 3. Run Ansible

```bash
cd ../ansible/
ansible all -m ping                         # quick connectivity test
ansible-playbook playbooks/test-all.yml     # full test suite
```

## 🔐 Security

The `terraform.tfvars` file contains the Proxmox root password in plain text. **It is excluded from Git** via `.gitignore`. For production use:

- Replace the `password` value with something strong
- Use a Proxmox API token instead of a password (I can adapt the config if needed)
- Store secrets in a vault (sops, Vault, etc.)

## 🔄 Teardown

```bash
cd terraform/
terraform destroy
```

## ⚠️ If image downloads fail

The URLs in `lxc.tf` and `vms.tf` point to Fedora 43. If the file no longer exists (new Fedora release):

- LXC template: go to `http://download.proxmox.com/images/system/` and look for the latest `fedora-*.tar.xz`
- Cloud image: go to `https://download.fedoraproject.org/pub/fedora/linux/releases/` for the latest Cloud Base release

Alternatively, download manually via the Proxmox UI:
- **CT Templates** → Templates tab → search for "fedora"
- **ISO Images** → download the Fedora cloud image from URL
